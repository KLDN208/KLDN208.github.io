Hexo + Github Pages 搭建个人博客

_：以下划线开头的单词为自定义，非专有名词；_confg.yml 除外

前置准备

  1. Node.js：不低于 10.13 recommend 不低于 12

  2. Git

  3. 安装 Hexo

  4. 创建项目

文件说明

  1. _config.yml：网站的配置信息

  2. package.json：项目依赖

    • 支持 EJS 和 markdown 语法

  3. scaffolds：脚手架｜模版

    • 当新建文章时，Hexo 会根据 scaffolds 建立文件

  4. source：资源文件夹

    • 存放用户资源的地方
    • 该文件夹下，只读取 _posts 文件下的文件

  5. themes：主题文件夹

    • Hexo 会根据指定的主题，生成对应主题的页面
    • 下载好主题文件后，在 _config.yml 文件中配置 theme:_themeName

  6. 基础配置 - _config.yml

    • 参数对应的值为 string 类型也可以不用引号包裹
    • ‘—‘：代表可用户自定义
    • 值为空的话可用 ‘ ‘ 或者不填代替
    • 字段与字段之间不允许用符号隔开
    • 参数名 参数含义 属性
      title 网站标题 必填
      subtitle 网站副标题 —
      description 网站描述 —
      keywords 网站关键词 —
      author 作者名字 —
      language 网站语言 默认：en
      可选：
      en - 英文
      zh-CN - 中文，包括繁体
      zh-Hans - 简体中文
      timezone 网站时区 默认：使用电脑的时区
      url 网站网址格式 必须以 http:// 或 https:// 开头
      root 网站根目录 —
      permalink 网站链接格式 默认：:year/:month/:day/:title/
      source_dir 资源文件夹 source
      pulic_dir 公共文件夹：
      用于存放 hexo d 后
      生成的站点文件      		 public
      tag_dir 标签文件夹 tags
      archive_dir 归档文件夹 archives
      category_dir 分类文件夹 categories
      date_format 日期格式 YYYY-MM-DD
      date_format 时间格式 HH:mm:ss
      theme 网站主题，Hexo 根据
      主题文件生成静态页面      		 默认：landscape
      禁用：false
      deploy 部署网站配置 type：方式 - git
      repo：代码部署地 - github.io 项目地址
      branch：分支 - master

  7. 启动服务

    • hexo server：默认运行于 http://localhost:4000

  8. 将 Hexo 项目部署到 Github Pages

    1. 在个人 Github 账户下，新建一个项目库 New repository

    2. Repository name 以 Owner 开头 .github.io 结尾

    3. Public / 可自行选择 / Create repository

    4. _config.yml 中 deploy 属性配置

      1. npm install hexo-deployer-git --save：加载该依赖，通过 hexo 命令 Git 方式将代码部署到 repo 指定地址
    5. 1.hexo clean：清除之前生成的站点文件 public
2.hexo generate：生成新的站点文件放于 public 中；可缩写：hexo g
3.hexo deploy：向指定地址部署代码；可缩写：hexo d

  9. Final Test

    • 进入 https://_repositoryName
    • 实例：https://xxx.github.io
